# Auth — 認證與授權領域

> **Bounded Context**：App\Domains\Auth
> **適合讀者**：後端開發者、安全工程師

## BC 職責說明

Auth BC 負責 AlleyNote 的身分驗證與權限控制，包含：

- **認證（Authentication）**：JWT（RS256）Token 發行、驗證、黑名單、刷新
- **授權（Authorization）**：5 策略組合的授權引擎（RBAC 角色 + 直接權限 + 屬性 + 自訂規則 + SuperAdmin 略過）
- **使用者管理**：使用者 CRUD、角色指派、密碼管理與安全性檢查
- **權限管理**：角色↔權限映射表維護、直接權限指派

**對應 HTTP 狀態碼**：認證失敗 401、授權失敗 403

## 依賴方向

```
Auth ←── Post（Session/權限查詢）
Auth ←── Application Middleware（JwtAuthenticationMiddleware、JwtAuthorizationMiddleware）
Auth ──→ Shared（ValueObjects）
Auth ──→ Firebase\JWT（第三方 JWT 函式庫）
```

## 對應 Middleware

- `JwtAuthenticationMiddleware` — 驗證 Access Token，回傳 401
- `JwtAuthorizationMiddleware` — 檢查權限，回傳 403
- `AuthorizationMiddleware` — 簡化授權介面
