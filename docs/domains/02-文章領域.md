# Post — 公告領域

> **Bounded Context**：App\Domains\Post
> **適合讀者**：後端開發者、內容管理者

## BC 職責說明

Post BC 是 AlleyNote 的核心領域，管理公告的完整生命週期：

- **公告 CRUD**：建立、編輯、發佈、軟刪除、回收
- **標籤管理**：標籤 CRUD、公告↔標籤關聯
- **內容處理**：CKEditor 富文本處理、XSS 消毒、內容審查
- **搜尋與篩選**：關鍵字搜尋、狀態篩選、分頁、標籤過濾
- **快取失效**：公告快取鍵管理與失效觸發
- **統計資料**：建立來源分佈、瀏覽次數增量

### Repositories（資料存取 — Facade + 3 Sub-Repos）

`PostRepository` 是 Facade，內部委派給三個子 Repository：

```
PostRepository（Facade／PostRepositoryInterface）
  ├── PostCrudRepository    — CRUD、分頁、軟刪除、標籤關聯
  ├── PostSearchRepository  — 標題搜尋、關鍵字搜尋
  └── PostAnalyticsRepository — 建立來源統計、來源分佈
```

| 類別 | 路徑 | 職責 |
|------|------|------|
| PostRepository | `Repositories/PostRepository.php` | Facade，實作 PostRepositoryInterface |
| PostCrudRepository | `Repositories/PostCrudRepository.php` | CRUD + 分頁 + 標籤 |
| PostSearchRepository | `Repositories/PostSearchRepository.php` | 搜尋 |
| PostAnalyticsRepository | `Repositories/PostAnalyticsRepository.php` | 建立來源分析 |
| TagRepository | `Repositories/TagRepository.php` | 標籤資料存取 |
| AdminPostReadRepository | `Repositories/AdminPostReadRepository.php` | 管理後台專用讀取 |
| PostBaseRepository | `Repositories/PostBaseRepository.php` | 共用基底 |
| HasTransactionSupport | `Repositories/HasTransactionSupport.php` | 交易支援 Trait |

## 依賴方向

```
Post ──→ Auth（查詢目前使用者 Session、權限）
Post ──→ Security（ActivityLogging 記錄操作）
Post ──→ Shared（CacheServiceInterface）
Post ──→ Statistics（公告資料為統計來源）
```
