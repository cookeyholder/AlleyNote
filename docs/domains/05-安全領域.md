# Security — 安全領域

> **Bounded Context**：App\Domains\Security
> **適合讀者**：後端開發者、安全工程師

## BC 職責說明

Security BC 統一管理 AlleyNote 的所有安全機制，涵蓋防護、偵測、日誌與速率限制：

- **XSS 防護**：HTML Purifier 整合，輸出消毒
- **CSRF 保護**：Token 驗證
- **安全標頭**：CSP、HSTS、X-Frame-Options 等 HTTP 標頭
- **活動日誌**：使用者操作審計日誌記錄與查詢
- **速率限制**：API 請求頻率控制（Redis 為後端）
- **IP 管理**：允許/拒絕清單規則管理
- **可疑活動偵測**：異常行為分析（失敗率尖峰、多使用者共享 IP）
- **機密管理**：敏感資訊儲存與存取
- **安全測試**：端點安全狀態檢查

## 對應 Middleware

- `CsrfMiddleware` — CSRF Token 驗證（HTTP 入口）
- `SecurityHeadersMiddleware` — 安全標頭注入
- `RateLimitMiddleware` — API 速率限制
- `PostViewRateLimitMiddleware` — 公告瀏覽速率限制

## 目前缺口

`RateLimitMiddleware` 直接依賴 `Infrastructure\Services\RateLimitService`，缺乏 Security BC 的抽象層。
應建立 `Domains\Security\Contracts\RateLimitServiceInterface` 讓 Middleware 依賴 Security BC 介面。

## 依賴方向

```
Security ──→ Shared（日誌、事件）
Security ←── Post（PostService 注入 LoggingSecurityService）
Security ←── Application Middleware（CsrfMiddleware、SecurityHeadersMiddleware）
```
