# SPA 架構總覽

## 概述

AlleyNote 前端為無建構工具的純 ES6 模組化 SPA，所有模組在瀏覽器端以 `import` / `export` 執行。入口檔案為 `frontend/index.html`，透過 `<script type="module">` 載入 `main.js`。

## ES6 Modules（無建構工具）

不使用 Webpack、Vite 等建構工具。所有 JS 檔案透過標準 ES module 語法管理相依：

```js
import { initRouter, router } from "./utils/router.js";
import { globalActions, globalGetters } from "./store/globalStore.js";
```

模組目錄：

```
frontend/js/
├── main.js           # 應用程式主入口
├── api/              # API 客戶端與模組
│   ├── client.js     # ApiClient 核心
│   ├── config.js     # API 設定
│   └── modules/      # 領域模組（auth, posts, roles, ...）
├── components/       # 可複用 UI 元件
├── layouts/          # 佈局元件（DashboardLayout）
├── pages/            # 頁面模組
│   ├── admin/        # 後台頁面
│   └── public/       # 公開頁面
├── store/            # 全域狀態管理
└── utils/            # 工具函式（router, notification, security, ...）
```

## SPA Router 流程

路由使用 **Navigo 8.x**（CDN 載入，掛在 `window.Navigo`）。

初始化流程：

1. `main.js` 呼叫 `initRouter()`
2. `initRouter()` 非同步等待 Navigo 載入完畢（polling 機制，最長 5 秒）
3. 建立 `new Navigo("/", { hash: false })` 實例
4. 註冊路徑與對應的 handler
5. 透過 `router.resolve()` 解析當前 URL

路由 handler 採用「延遲載入」模式：

```js
router.on("/admin/posts", async () => {
  if (await requireAuth()) {
    const module = await import("../pages/admin/posts.js");
    await module.renderPostsList();
  }
});
```

路由守衛（Route Guards）：

- `requireAuth()` — 未認證者導向 `/login`
- `guestOnly()` — 已認證者導向 `/admin/dashboard`

路由表（部分）：

| 路徑 | 頁面 | 守衛 |
|------|------|------|
| `/` | 首頁 | — |
| `/login` | 登入 | guestOnly |
| `/posts/:id` | 文章內頁 | — |
| `/admin/dashboard` | 後台儀表板 | requireAuth |
| `/admin/posts` | 文章管理 | requireAuth |
| `/admin/posts/create` | 新增文章 | requireAuth |
| `/admin/posts/:id/edit` | 編輯文章 | requireAuth |
| `/403` | 403 錯誤 | — |
| `/500` | 500 錯誤 | — |
| `notFound` | 404 頁面 | — |

## Global Store

位於 `frontend/js/store/globalStore.js`，採用簡單的 Observer 模式。

核心類別 `GlobalStore`：

```js
class GlobalStore {
  state = { user: null, isAuthenticated: false, loading: false, error: null }
  listeners = []

  getState()       // 取得狀態快照
  setState(updates) // 更新狀態 + 通知監聽者
  subscribe(listener) // 訂閱變化，回傳取消函數

  setUser(user)    // 設定使用者 + 寫入 localStorage
  clearUser()      // 清除使用者 + localStorage
}
```

匯出介面：

- `globalGetters` — `getUser()`、`isAuthenticated()`、`isAdmin()`、`getState()`
- `globalActions` — `setUser()`、`clearUser()`、`setLoading()`、`restoreUser()`

狀態初始化時（`restoreState()`）透過檢查 Cookie 中的 `access_token` 或 `auth_mode=cookie` 判斷是否已認證，不需額外 API 呼叫。

## API Client 模組結構

兩層結構：

1. **ApiClient**（`api/client.js`）— 核心 HTTP 客戶端，封裝 `fetch`
2. **API Modules**（`api/modules/*.js`）— 各領域的操作封裝，回傳 Promise

ApiClient 特性：

- 統一 `baseURL`（`/api`）、`timeout`（30s）、`Content-Type`
- 自動附加 `Authorization: Bearer` 與 `X-CSRF-TOKEN`
- Token 刷新（401 時自動透過 `/auth/refresh` 取得新 token）
- CSRF 失效自動重試
- 統一錯誤處理（`ApiError` 類別）
- `silent` 選項控制是否顯示錯誤通知

API Modules：

```js
import { postsAPI } from "../api/modules/posts.js";
const posts = await postsAPI.list({ page: 1 });
await postsAPI.create({ title, content, status });
await postsAPI.update(id, { title, content });
await postsAPI.delete(id);
```

## Tailwind CDN + CKEditor 5

### Tailwind CSS

透過 CDN 載入並自訂設定：

```html
<script src="https://cdn.tailwindcss.com"></script>
<script>
  tailwind.config = {
    theme: {
      extend: {
        colors: { modern: { ... }, accent: { ... } },
        fontFamily: { inter: ["Inter", "sans-serif"] },
      },
    },
  };
</script>
```

自訂顏色系統：`modern`（灰階中性色）、`accent`（品牌重點色）。

### CKEditor 5

載入方式：

```html
<link rel="stylesheet" href="https://cdn.ckeditor.com/ckeditor5/43.3.1/ckeditor5.css">
<script src="https://cdn.ckeditor.com/ckeditor5/43.3.1/ckeditor5.umd.js"></script>
```

封裝元件 `frontend/js/components/RichTextEditor.js` 提供 `initRichTextEditor()`、`getRichTextEditorContent()`、`destroyRichTextEditor()` 等 API。另有 `CKEditorWrapper` 類別（`components/CKEditorWrapper.js`）用於 class-based 頁面整合。

目前用於「系統設定」頁面的頁腳描述欄位（`frontend/js/pages/admin/settings.js`）。

### Chart.js

透過 CDN 載入 `chart.js@4.4.0`，用於統計頁面的流量趨勢圖與登入失敗分析表。

### DOMPurify

透過 CDN 載入 `dompurify@3.0.6`，用於 XSS 防護。

## 主入口初始化流程

`frontend/js/main.js` 的 `initApp()` 啟動順序：

```
DOMContentLoaded
  → show loading indicator
  → globalActions.restoreUser()   (從 Cookie 恢復認證狀態)
  → if authenticated: authAPI.me() (驗證 token 有效性)
  → initRouter()                  (建立路由、註冊 handler、解析 URL)
  → setupGlobalListeners()        (auth:logout, online/offline, error)
  → hide loading indicator
```
