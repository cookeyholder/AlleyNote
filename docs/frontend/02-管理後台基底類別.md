# 管理後台基底類別

## 概述

`BaseAdminPage`（`frontend/js/components/BaseAdminPage.js`）是管理後台 class-based 頁面的基底類別，封裝共用的生命週期與工具方法。

## 生命週期（模板方法模式）

```
constructor → init() → loadData() → render() → attachEventListeners() → afterRender()
```

實作細節（`BaseAdminPage.js`）：

```js
export default class BaseAdminPage {
  constructor() {
    this.loading = false;
  }

  async init() {                // 模板方法，子類別不應覆寫
    this.showLoading();
    try {
      await this.loadData();    // ① 載入資料
    } finally {
      this.hideLoading();
    }
    await this.render();              // ② 渲染 DOM
    this.attachEventListeners();      // ③ 綁定事件
    this.afterRender();               // ④ 渲染後處理（選擇性）
  }

  async loadData() {}           // 子類別覆寫
  async render() {}             // 子類別覆寫（必要）
  attachEventListeners() {}     // 子類別覆寫（必要）
  afterRender() {}              // 子類別選擇性覆寫
}
```

子類別不需要手動管理初始化順序，只需實作各自的資料載入、渲染與事件綁定邏輯。

## 5 個子頁面繼承關係

| 頁面類別 | 檔案路徑 | loadData | afterRender |
|---------|---------|----------|-------------|
| `StatisticsPage` | `pages/admin/statistics.js` | 載入 overview / popular / loginFailures / trafficData | 初始化 Chart.js 圖表 |
| `RolesPage` | `pages/admin/roles.js` | 載入角色列表 + 權限列表（Promise.all） | — |
| `ProfilePage` | `pages/admin/profile.js` | 載入當前使用者資料 | — |
| `UsersPage` | `pages/admin/users.js` | 載入使用者列表 + 角色列表 | — |
| `TagsPage` | `pages/admin/tags.js` | 載入標籤列表 | — |

各頁面皆透過靜態 wrapper 函數啟動生命週期：

```js
// 範例：statistics.js
export async function renderStatistics() {
  const page = new StatisticsPage();
  await page.init();
}
```

## 共用方法

### showLoading() / hideLoading()

設定 `this.loading = true/false`，子類別在 `render()` 中可依據此旗標決定顯示載入畫面或內容。

```js
// StatisticsPage 的 render 方法
render() {
  const content = this.loading ? this.renderLoading() : this.renderContent();
  renderDashboardLayout(content, { title: "系統統計" });
  bindDashboardLayoutEvents();
}
```

### escapeHtml(str)

委派至 `utils/security.js` 的 `escapeHtml()`，用於防止 XSS 攻擊：

```js
this.escapeHtml(post.title)
// 輸出：&amp; &lt; &gt; &quot; &#039;
```

轉換表：

| 字元 | 轉換 |
|------|------|
| `&` | `&amp;` |
| `<` | `&lt;` |
| `>` | `&gt;` |
| `"` | `&quot;` |
| `'` | `&#039;` |

## 與非 class-based 頁面的差異

部分管理頁面仍使用「函數式寫法」（如 `posts.js`、`postEditor.js`、`settings.js`、`dashboard.js`），不使用 `BaseAdminPage`。這些頁面直接透過 `renderDashboardLayout()` 渲染，各自管理自己的狀態（module-level 變數）。

```
BaseAdminPage 子類別：StatisticsPage, RolesPage, ProfilePage, UsersPage, TagsPage (5 頁)
函數式寫法：posts, postEditor, settings, dashboard (4 頁)
```
