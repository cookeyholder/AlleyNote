# 文件治理規範

## 1. 何時必須更新文件

當 PR 變更以下內容時，必須同步更新文件：

- 執行時行為（API／UX／授權／狀態流程）
- 會影響操作或開發流程的相依套件版本
- CI/CD 工作流程、品質關卡或發佈／部署程序
- 安全控制或設定（驗證、CSRF、標頭、機敏資訊處理）
- 貢獻者或維運人員依賴的架構契約

若判斷不需要更新文件，PR 說明必須附上簡短理由。

## 2. 文件審查檢查清單

審查者應確認：

- [ ] 範圍完整：受影響主題已反映到 canonical 文件
- [ ] 內容正確：指令、路徑、連接埠符合目前儲存庫行為
- [ ] 連結完整：更新後連結都能對到現有檔案
- [ ] 去重一致：主動態文件不存在彼此衝突的重複說明
- [ ] 歸檔整潔：被取代文件已歸檔或移除，並附合理說明

## 3. OpenSpec 與文件同步規則

對每個進行中的 change：

1. proposal／design／tasks 必須標示受影響的 canonical 文件。
2. implementation 完成前，必須先更新對應文件。
3. 最終驗證必須包含受影響文件的連結／路徑檢查。

## 4. Canonical 入口清單

- 根目錄概覽：`README.md`
- 文件總索引：`docs/INDEX.md`
- 領域與 API 詳細內容：`docs/api/README.md` 與其連結的 canonical runbook
- 歷史文件：`docs/archive/`
