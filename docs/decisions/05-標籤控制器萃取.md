# ADR-5: 標籤控制器內嵌 HTML/JS 萃取

**狀態**: 已完成
**日期**: 2026-07-15

## 背景

`TagManagementController`（702 行）在原始設計中將 HTML 樣板與大量的內嵌 JavaScript 直接以 PHP 字串拼接方式置於 `getTagPageHtml()` 方法（約 674 行，行 705–1378）。這種做法導致：

1. **前後端緊密耦合**：HTML 結構與 JavaScript 邏輯埋藏在 PHP Controller 中，修改前端樣式需修改 PHP 檔案
2. **無法利用前端工具**：無語法高亮、無 Prettier 格式化、無模組化 JS
3. **CSP nonce 難以管理**：透過 `str_replace('CSP_NONCE_PLACEHOLDER', $nonce, $html)` 替換，容易出錯
4. **Controller 職責混雜**：Controller 同時處理 HTTP 請求、HTML 渲染與客戶端互動邏輯

## 決策

將 HTML 樣板萃取至獨立的 PHP 視圖檔案，JavaScript 萃取至獨立的 ES6 模組：

- `backend/resources/views/admin/cache/tags.php`：完整 HTML 結構（DOCTYPE、head、Bootstrap CSS、navbar、統計卡片、表格、Modals）
- `frontend/js/pages/admin/cacheTags.js`：`CacheTagsPage` 類別封裝所有客戶端邏輯（資料載入、表格渲染、分頁、搜尋、Modal、確認對話框）

Controller 的 `renderTagPage()` 改為：
```php
$nonce = $this->headerService->generateNonce();
require __DIR__ . '/../../../../resources/views/admin/cache/tags.php';
```

CSP nonce 透過 `<?= $nonce ?>` 直接輸出，JS 模組以 `<script type="module" nonce="<?= $nonce ?>" src="/js/pages/admin/cacheTags.js">` 載入。

## 結果

**變更檔案**：
- 新增：`backend/resources/views/admin/cache/tags.php`（HTML 視圖）
- 新增：`frontend/js/pages/admin/cacheTags.js`（ES6 模組，503 行）
- 修改：`TagManagementController.php` — 移除 `getTagPageHtml()`（-674 行），`renderTagPage()` 改為 require 視圖

**對比**：
- **Before**：Controller 702 行，HTML+JS 內嵌於 PHP 字串
- **After**：Controller 約 28 行（僅 HTTP 邏輯），視圖獨立，JS 獨立模組

**測試**：靜態分析通過，手動巡覽 `/admin/cache/tags` 確認統計卡片、表格、分頁、搜尋、Modal 正常運作，CSP 無違規。

## 替代方案

1. **維持內嵌**：不萃取 — 拒絕原因：違反關注點分離，Controller 職責過重。
2. **使用 PHP 樣板引擎（Twig/Smarty）**：引入外部樣板引擎 — 拒絕原因：專案為自製框架，無此依賴，引入成本高。
3. **全端 SPA 元件化**：將此頁面改為完全由前端渲染 — 拒絕原因：需額外 API 端點，改動範圍過大。

## 取捨

- 視圖檔案使用 `<?= $nonce ?>` 嵌入 CSP nonce，仍需在 PHP 環境中渲染
- JS 模組透過 `window.bootstrap` 全域存取 Bootstrap JS，未使用 npm 套件管理
