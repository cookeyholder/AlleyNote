# ADR-3: IP 邏輯統一至 NetworkHelper

**狀態**: 已完成
**日期**: 2026-07-15

## 背景

系統中 6 個呼叫者各自實作了不同的 IP 萃取邏輯，導致重複與不一致：

| 呼叫者 | 自實作方法 | 行數 |
|--------|-----------|------|
| `JwtAuthorizationMiddleware` | `getClientIpAddress()`、`isIpInList()`、`ipMatches()` | ~80 行 |
| `RateLimitMiddleware` | `getRealClientIP()` | ~30 行 |
| `PostViewRateLimitMiddleware` | `getRealClientIP()` | ~30 行 |
| `StatisticsAdminController` | `getClientIpAddress()` | ~20 行 |
| `CSPReportController` | `getClientIP()` | ~20 行 |
| `DeviceInfo`（ValueObject） | `maskIpAddress()` | ~15 行 |

這些實作在標頭優先順序、私有 IP 過濾邏輯、Fallback 行為上存在細微差異，可能導致不同端點對同一請求判斷出不同 IP。

## 決策

將所有 IP 萃取邏輯集中至 `NetworkHelper`，擴充其公共靜態方法：

- `getClientIpFromServerParams()`：從 `$serverParams` 依優先順序萃取 IP
- `getClientIpWithPrivateCheck()`：信任代理時的私有 IP 過濾版本
- `maskIpAddress()`：IP 遮罩（從 `DeviceInfo` 搬遷）
- `isIpInRanges()` 與 `ipInNetwork()`：改為 `public static`，支援 CIDR 與萬用字元比對

原有 `getClientIp()` 與 `getTrustedProxies()` 簽章保持不變。

## 結果

**變更檔案**：
- 修改：`Shared/Helpers/NetworkHelper.php` — 新增 3 個公開方法，2 個方法改為公開
- 修改：`JwtAuthorizationMiddleware` — 內聯實作改為委派
- 修改：`RateLimitMiddleware` — 內聯實作改為委派
- 修改：`PostViewRateLimitMiddleware` — 內聯實作改為委派
- 修改：`StatisticsAdminController` — 內聯實作改為委派
- 修改：`CSPReportController` — 內聯實作改為委派
- 修改：`DeviceInfo` ValueObject — `maskIpAddress` 改為委派

**對比**：
- **Before**：6 處各自實作，約 200 行重複程式碼，行為細節不一致
- **After**：1 處統一的 NetworkHelper，約 245 行，所有呼叫者委派至此

**測試**：撰寫 NetworkHelper 完整單元測試（IPv4/IPv6、CIDR、萬用字元、標頭優先順序、私有 IP 過濾、邊界案例），並以行為比對測試矩陣驗證各呼叫者取代前後輸出一致。

## 替代方案

1. **Middleware 基底類別**：建立 `AbstractIpAwareMiddleware` — 拒絕原因：非所有呼叫者都是 Middleware（Controller、ValueObject 也需要）。
2. **獨立 IpExtractor Service**：建立 `IpExtractionService` 領域服務 — 拒絕原因：IP 萃取純屬基礎設施關注點，不應導入領域層。
3. **保留各自實作**：不統一 — 拒絕原因：重複程式碼難以維護，行為不一致造成安全漏洞。

## 取捨

- 統一行為後，若某個呼叫者需要特殊 IP 處理邏輯，需透過參數化達成（如 `$fallback`、`$iterateAllIps`、`$filterFlags`），無法完全客製
- NetworkHelper 為靜態工具類別，無法輕易 mock，測試採用行為比對策略
