# ADR-6: 管理後台頁面基底類別

**狀態**: 已完成
**日期**: 2026-07-15

## 背景

前端管理後台的 5 個頁面（`profile.js`、`roles.js`、`statistics.js`、`tags.js`、`users.js`）各自獨立實作了類似的生命週期與工具方法：

1. 初始化流程：`init()` → 載入資料 → 渲染 → 綁定事件監聽器
2. 載入狀態管理：`showLoading()` / `hideLoading()`
3. HTML 跳脫：各自的 `escapeHtml()` 實作

這些重複樣板導致：
- 新增管理頁面需複製貼上大量樣板程式碼
- 修改生命週期行為需逐一更新所有頁面
- `escapeHtml` 重複實作 5 次

## 決策

建立 `BaseAdminPage` 基底類別，封裝共用生命週期與工具方法：

- 模板方法模式：`init()` 定義固定流程，子類別覆寫 `loadData()`、`render()`、`attachEventListeners()`、`afterRender()`
- 載入狀態管理：統一的 `showLoading()` / `hideLoading()`
- `escapeHtml`：從 `../../utils/security.js` 匯入，統一跳脫邏輯

同步建立 `api/modules/tags.js` API 模組，參照現有 `posts.js`、`users.js` 模式，封裝標籤 CRUD。

## 結果

**變更檔案**：
- 新增：`frontend/js/components/BaseAdminPage.js`（85 行基底類別）
- 新增：`frontend/js/api/modules/tags.js`（標籤 API 模組）
- 修改：`profile.js` — extends `BaseAdminPage`，移除重複樣板
- 修改：`roles.js` — extends `BaseAdminPage`，移除重複樣板
- 修改：`statistics.js` — extends `BaseAdminPage`，移除重複樣板
- 修改：`tags.js` — extends `BaseAdminPage`，改用 `api/modules/tags.js`
- 修改：`users.js` — extends `BaseAdminPage`，移除重複樣板

**對比**：
- **Before**：5 個頁面各自實作生命週期（~30 行重複/頁），各自 escapeHtml（~5 行重複/頁）
- **After**：基底類別統一生命週期與工具方法，每個頁面僅保留特有邏輯

**測試**：前端 lint 通過，手動巡覽 5 個管理頁面確認功能正常。

## 替代方案

1. **維持各自實作**：不萃取基底類別 — 拒絕原因：重複樣板持續累積技術債。
2. **使用 Vue/React 等框架**：導入前端框架管理生命週期 — 拒絕原因：專案為純 ES6 模組化 SPA，不引入框架。
3. **Mixin 模式**：撰寫可混入的工具物件 — 拒絕原因：ES6 模組無原生 mixin 支援，基底類別繼承更直觀。

## 取捨

- 使用類別繼承（`extends`）而非組合（composition），若頁面需完全不同的生命週期流程，需調整基底類別設計
- `escapeHtml` 從外部模組匯入，新增依賴關係但避免了重複實作
