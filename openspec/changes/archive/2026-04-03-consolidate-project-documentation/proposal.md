## 為何要做

在合併多個 PR（包含安全強化與多個 Dependabot 更新）後，專案文件已出現分散且部分過時的情況。需要建立一致且可維護的文件結構，讓貢獻者能快速取得正確的安裝、架構、CI 與維運指引。

## 變更內容

- 重整並整併儲存庫文件，收斂為更精簡且清楚的一組 canonical 檔案。
- 更新 `README.md`、`docs/` 與維運 runbook 中過時內容，與目前程式碼與流程對齊。
- 建立清楚的責任邊界與更新規則，確保後續 PR 能維持文件與行為同步。
- 歸檔或移除不再具有獨立維運價值的重複報告／版本文件。
- 新增輕量文件索引與交叉連結，避免指令重複散落。

## 能力項目

### 新增能力
- `documentation-information-architecture`：定義 canonical 文件架構、整併規則與唯一可信來源映射。
- `documentation-maintenance-governance`：定義文件更新觸發條件、審查清單與品質關卡。

### 修改能力
- `dev-environment`：更新需求與參考文件，讓本機安裝／執行／測試流程統一導向 canonical 文件。

## 影響範圍

- 影響區域：`README.md`、`docs/**`、部分 `openspec/` 文件與貢獻流程相關參考。
- 不涉及正式環境執行時行為變更。
- 可降低文件數量與重複維護成本。
- PR 流程影響：文件更新檢查清單成為交付的一部分。
